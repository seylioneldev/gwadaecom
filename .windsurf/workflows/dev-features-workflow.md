Tu suis ce workflow pour m'aider à développer une nouvelle fonctionnalité dans le projet gwadaecom (Next.js + Stripe + Firestore + Firebase Auth).

1) Comprendre
- Reformule ma demande en 1–2 phrases.
- Pose au maximum 3 questions si quelque chose est flou.

2) Analyse minimale
- Identifie les fichiers les plus probables.
- Lis seulement ces fichiers (pas de scan global du repo sauf si je le demande).

3) Plan
- Propose un plan en 3–5 étapes courtes.
- Attends ma validation si le changement touche : checkout, Stripe, Firestore ou l’admin.

4) Implémentation
- Applique le plan par petites étapes.
- Pour chaque étape, indique les fichiers modifiés et le but.
- Respecte le style de code existant, pas de gros refactor si ce n’est pas nécessaire.

5) Vérification
- Propose des tests manuels simples spécifiques à gwadaecom (ex : scénario checkout, panier, admin).
